# EROL Public Scope

## Purpose

This document defines what the public EROL repository is meant to be.

## Public Scope

The public EROL repository is intended to show:

- research identity
- architecture direction
- public-safe signal framing
- observability logic
- proportionality-focused scope

## Not Public By Default

The public EROL repository is not meant to expose:

- private implementation logic
- politically sensitive operational playbooks
- internal deployment assumptions
- private data pipelines
- internal calibration details

## Public Role

This repository should function as:

- a public technical window
- a partner-safe observability surface
- a neutral architecture layer for discussion

It should not function as the full private project space.
