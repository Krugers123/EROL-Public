# EROL Public Signal Model

## Purpose

This document explains the public-safe signal families behind EROL.

## Core Signal Families

### Capacity Signals

Used to track:

- hospital load
- ICU pressure
- saturation trends
- healthcare strain

### Severity Signals

Used to track:

- mortality
- hospitalization trends
- escalation intensity

### Intervention Signals

Used to track:

- restrictions
- mandates
- policy shifts
- response timing

### Behavioral Signals

Used to track:

- mobility changes
- contact proxies
- compliance indicators
- behavioral response shifts

### Reliability Signals

Used to track:

- testing bias
- reporting lag
- data refresh delay
- source disagreement

## Public Reading

The point of EROL is not to elevate one signal family above all others.

The point is to observe whether:

- signal families converge
- signal families diverge
- intervention trajectories remain proportional to the combined signal field

## Working Rule

EROL becomes more useful when:

- multiple signal families are read together
- proportionality is evaluated over time
- drift is treated as a trajectory problem, not a headline problem
